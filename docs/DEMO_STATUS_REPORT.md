# Научный клубок — отчёт о готовности к защите

_Дата: 2026-07-04. Ветка `main`. Отчёт сформирован по итогам многоагентного анализа (100 агентов — разбор проекта, 23 — консолидация фронта, 36 — аудит соответствия task-2, 48 — демо-хардениг)._

---

## 1. Что сделано в этой сессии

### 1.1. Консолидация интерфейса под жюри (86 → 25 разделов)

Сайдбар сокращён с **86 до 25 пунктов** в 9 продуктовых группах — интерфейс читается как инструмент учёного, а не как инженерная панель. Все удалённые экраны **остались в коде** (роуты и компоненты живы), убраны только из бокового меню — вернуть = одна строка.

**5 новых вкладочных хабов** (общий компонент `TabHub`) свернули 17 почти-дублирующих экранов:

| Хаб | Вкладки |
|---|---|
| Сущности и похожие | Карточка сущности · Похожие · История сущности |
| Инспектор доказательств | Цепочка доверия · Цитата на странице · (Фигуры/Подписи — скрыты) |
| Доверие к источникам | Доверие к источнику · Происхождение цитат |
| Курирование | Очередь ревью · Дубликаты · Откат слияний · Правка таблиц · Что изменилось |
| Ход мысли ассистента | Ход мысли · Разбор по шагам · Воспроизводимость |

Ранее в ту же логику свели: «Поиск по корпусу» (фасеты + диапазоны + глоссарий), «Запрос к графу» (поглотил чат), «Карта пробелов» (кэш агентного потока), «Обзор базы знаний» (встроены покрытие по доменам, матрица материал×свойство и WebGL-клубок).

**Скрыто из меню (53 инженерных экрана):** приём/конвейеры/линидж/ML-NER, весь блок eval/quality/gate/benchmark, внутренности агента, governance, i18n. Группы «Данные» и «Качество» убраны целиком.

### 1.2. Чистка текстов от инженерного жаргона (169 правок, 33 файла)

Видимые пользователю строки переписаны так, чтобы говорить, **что** получает пользователь, а не **как** это устроено. Убраны: имена моделей (`glm-5.2`, `deepseek`), слово «агент/агент-приоритизатор», названия движков (Neo4j/Qdrant/Cypher/Sigma/Dagster), ссылки на §-разделы, сырые метрики (F1/NLI/recall@k).

> Было: «Каждый пробел знаний оценивает отдельный агент-приоритизатор (glm-5.2, до 10 параллельно): важность × осуществимость…»
> Стало: «Пробелы в знаниях, отсортированные по важности и выполнимости — с конкретным следующим шагом по каждому».

## 2. Аудит соответствия task-2 (36 агентов, адверсариальная проверка)

Проверялась **работающая** система (весь путь: фронт → api → смонтированный роутер → стор → реальные данные или заглушка), а не наличие файлов.

### Итог: готовность ≈ **58 %**. Реально реализовано И подключено: **MUST 5/16, PLUS 2/9**.

- **Implemented: 11, Partial: 23, Missing: 1** (из 35 требований).
- Полностью рабочие MUST: **M1** (запрос → ответ с цитатами), **M3** (многопараметрический запрос), **M13** (визуализация графа), **M14** (загрузка новых документов), **M16** (юзабилити) + **A10** (уровень уверенности).

### 2.1. Что честно работает — это и есть питч

- Вопрос на естественном языке → **структурированный ответ со ссылками на источники и мерой уверенности** за секунды, без Cypher/CLI.
- Клик по цитате → **подсвеченный фрагмент источника + номер страницы**.
- Граф `материал → процесс → оборудование → результат` + WebGL-«клубок» всего корпуса.
- Многопараметрический запрос (материал + процесс + числовое условие + география + год) реально сужает факты графа.
- Сравнение технологий с пометкой пробелов и экспортом CSV.
- **Живая загрузка PDF/DOCX/XLSX** → подграф, влитый в тот же стор, который опрашивает агент.
- Рекомендации «куда направить исследование дальше» (ранжированный бэклог пробелов + следующий шаг).
- RU/EN + единицы измерения к одной сущности (словарь 218 терминов).
- Ручная правка графа с аудит-следом + Evidence Pack (ZIP/HTML/PDF) с SHA-256-манифестом воспроизводимости.

### 2.2. Два самых опасных места (а судят по корректности/верифицируемости в первую очередь)

- **A1 — весь показ работает на ~83-узловом ручном сиде** (`packages/kg_retrievers/src/kg_retrievers/seed.py`, авто-загрузка в `apps/api-gateway/.../deps.py`), заточенном ровно под 4 вопроса. Вне сценария → «Достаточных структурированных фактов не найдено»; загруженные документы извлекаются только правилами, без переиндексации вектора. «Работает на вашем корпусе / <5 с на 1М сущностей» защитить нельзя (масштаб де-скопнут в ADR-0005, теста на большом графе нет).
- **A6 — гард от галлюцинаций отключён на геройском стриминг-пути** (`answer_query_stream` не вызывает `apply_verification`), а где работает — проверяет лишь наличие маркера `[n]`, а не совпадение числа с источником. Неверное-но-цитированное число пройдёт как «проверено».

### 2.3. Видимые поломки, которые заметит жюри (все — быстрый фикс)

| ID | Симптом | Правка |
|---|---|---|
| M11 | Арбитр противоречий показывает «мало данных» с пустыми карточками — узел :Contradiction в сиде не связан с двумя измерениями (0.2 vs 0.5 м/с) | S |
| P6 | «Утвердить» в курировании пишет в аудит действие **REJECT** вместо ACCEPT | S |
| M7 | Вкладки «Фигуры/Подписи» → HTTP 503 (PyMuPDF не в зависимостях) + пустой селектор | S |
| M4 | Вкладка «Диапазоны» рисует пустую гистограмму (в сиде нет Parameter/HAS_PARAMETER) | S |
| M9 | Чипы даты/года в цитатах пустые — у seed-Evidence нет source_date/source_year | S |

### 2.4. Чего НЕ заявлять жюри

Полный геол.→металлург. цикл (есть только металлургия) · Патенты/места внедрения как сущности (патент = тег документа) · 5-way таксономия пробелов / 5-facet география / 4 раздельные даты (нет как специфицировано) · OCR / zh-es · техно-экономика (Admin показывает только счётчик узлов) · рекомендация экспертов/лабораторий (dead code) · экспорт RIS/BibTeX (unreachable).

## 3. Демо-хардениг — **применено** (коммит `dc357db`)

Воркфлоу из 48 агентов: авторинг точных патчей + адверсариальная проверка каждого (4 скептика на фикс, т.к. локально нет сборки) + свип на доп. поломки от сида → сведённый манифест. Применены только патчи, где автор + большинство скептиков (≥3/4) сошлись и `old` совпал дословно.

**Применённые фиксы (все видимые поломки из §2.3 + пара MUST):**

| ID | Что исправлено | Файл |
|---|---|---|
| M11 | Узел :Contradiction связан с обоими измерениями (0.2 vs 0.5 м/с) → арбитр показывает реальные карточки сторон, а не «мало данных» | `seed.py` |
| M9 | seed-evidence получает детерминированный `source_year`/`source_date` → чипы года и «актуал.» в цитатах заполняются | `seed.py` |
| P6 | Курирование нормализует `approved`→`accepted` → аудит пишет ACCEPT, а не REJECT | `curation-service/curation.py` + `CurationView.tsx` |
| M7 | Убраны вкладки «Фигуры/Подписи» (HTTP 503, нет PyMuPDF) | `EvidenceInspectorHubView.tsx` |
| M4 | Убрана пустая вкладка «Диапазоны» (в сиде нет Parameter) | `CorpusExplorerView.tsx` |

| +свип | Дашборд-покрытие без 2 всегда-пустых доменов; плейсхолдер «Панели рисков» переведён на данные сида; скрыта мёртвая вкладка «История сущности»; убран мёртвый фильтр «Свойства» | `admin.py`, `WarningPanelView.tsx`, `GraphExploreView.tsx`, `EntityDetailView.tsx` |

**Отложено под ручную проверку (behavior-changing / нужен реальный прогон — тут не собрать):**
- **A6** — подключить `apply_verification` в стрим-путь `answer_query_stream` + сверка значения числа с источником (усилие M). Все 4 скептика проголосовали «apply», но правило манифеста увело high-risk/needs_test в ручную проверку — это меняет поведение геройского пути, а протестировать локально нельзя.
- **A1** — пред-загрузить 2-3 реальных PDF Норникеля + переиндексировать вектор, чтобы вопрос вне 4 засеянных возвращал реальные факты (усилие L).
- Досев сравнения (RO/IE/ED колонки) и +2 демо-Gap для «Карты пробелов» — усилие M, второй ещё и дёргает live-LLM-оценку при каждом открытии экрана (латентность) → намеренно не применял.

> ⚠️ Сборку (`npm run build` / `tsc`) в этом окружении запустить нельзя (нет Node/venv). Все патчи проверены статически (совпадение `old`, отсутствие висячих импортов, соответствие типам `AnswerTable`/`Citation`). **Обязательно соберите фронт перед защитой.**

## 4. Демо-дисциплина на защите

1. Держаться **4 засеянных вопросов** (десалинация / циркуляция католита никеля / распределение Au-Ag-МПГ штейн-шлак / закачка шахтных вод).
2. Не открывать экраны фигур / OCR / техно-экономики / экспертов.
3. Вести историю: **цитата-подсветка → уверенность → сравнение → живая загрузка документа**.
4. На числах честно: показанные значения — реальные stored-значения сида, риска «неверного числа» на сценарии нет.
