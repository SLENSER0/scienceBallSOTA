# Neo4j (server profile) — §3.9 / §3.10-3.13

The default runtime is the **embedded Kuzu** profile (ADR-0005). This directory
provisions Neo4j for the server profile.

## Plugins

`NEO4J_PLUGINS='["apoc","graph-data-science"]'` (see `infra/docker-compose.yml`).
Verify: `cypher-shell < verify_plugins.cypher` → APOC + GDS versions.

## Migrations

Generated by `make schema-gen` into `migrations/`:
- `0001_constraints.cypher` — uniqueness constraints for every label (§3.10)
- `0002_indexes.cypher` — range/property indexes (§3.11)
- `0003_fulltext.cypher` — fulltext entity/evidence indexes (§3.12)
- `0004_vector.cypher` — 384-dim vector index on `:Entity` (§3.13)

Apply in order:
```bash
for f in migrations/000*.cypher; do cypher-shell -u neo4j -p password < "$f"; done
```

## Seed

`python infra/neo4j/seed/seed_graph.py` builds the demo graph (idempotent;
works against the embedded store by default — for Neo4j, point a Neo4j-backed
`GraphStore` adapter at it).

## Backup / restore

```bash
neo4j-admin database dump   neo4j --to-path=/backups     # backup
neo4j-admin database load   neo4j --from-path=/backups --overwrite-destination=true
```

Embedded profile: back up the `var/` directory (`scripts/backup.sh`).

## Read-only agent user (§19.6)

The agent/graph read path connects as a dedicated `reader` role; writes go only
through curation-service as `writer`. Create on first boot:

```cypher
CREATE USER kg_reader SET PASSWORD 'change-me' CHANGE NOT REQUIRED;
GRANT ROLE reader TO kg_reader;      -- built-in read-only role
CREATE USER kg_writer SET PASSWORD 'change-me' CHANGE NOT REQUIRED;
GRANT ROLE editor TO kg_writer;
```

Defense-in-depth on top of the app-level `cypher_guard` (static mutating-clause
rejection + LIMIT + allowlist) and `ALLOW_RAW_CYPHER=false` default.
